# 기초시스템프로그래밍 실습/출제 코드 정리

이 저장소는 강의 PDF/PPTX 자료에 들어있는 **실습문제**와 **출제문제**를
학생이 직접 작성한 것 같은 C 코드로 옮겨 장별로 정리하는 곳이다.

## 작업 흐름

사용자가 PDF/PPTX 자료의 위치나 이름을 알려주면:

1. 해당 자료에서 텍스트를 추출해 내용을 읽는다. (아래 "자료 읽는 법")
2. 자료를 **실습문제 파트**와 **출제문제 파트**로 구분한다.
3. 각 문제를 하나씩 C 파일로 옮긴다.
4. 해당 장 폴더의 `실습/` 또는 `출제/` 하위폴더에 넣는다.

여러 문제가 한 자료에 있으면 문제별로 파일을 따로 만든다.
어느 파트인지(실습/출제) 자료에서 애매하면 사용자에게 확인한다.

## 자료 읽는 법 (OCR/텍스트 추출)

이 환경에는 `poppler-utils`(pdftoppm/pdftotext)와 `tesseract`가 없다.
대신 **PyMuPDF(`fitz`)** 가 설치돼 있어 이것으로 처리한다.

- **PDF**: `python3 -c "import fitz; ..."` 로 `get_text()` 추출.
  강의/실습 PDF 대부분이 PPT를 내보낸 것이라 텍스트가 그대로 박혀 있어
  별도 OCR이 필요 없다. (한글 띄어쓰기가 붙어 나오는 건 정상)
- **PPTX**: `zipfile`로 열어 `ppt/slides/slideN.xml` 에서
  `<a:t>...</a:t>` 텍스트를 정규식으로 뽑는다. (python-pptx는 없음)
- **이미지로만 된 페이지**(텍스트가 거의 안 나올 때): `fitz`로 해당 페이지를
  PNG로 렌더링한 뒤 Read 도구로 그 PNG를 읽어(비전) 코드를 옮긴다.

## 폴더 / 파일 이름 규칙

장 폴더는 **작업 루트(`/mnt/e/dev/2026_c/`)에 바로** 만든다.
`기초시스템프로그래밍(윤석훈)/` 안에는 넣지 않는다. (그 폴더는 원본 자료 보관용)

```
<N장 제목>/
  실습/
    labNN_MM.c     # NN = 장 번호(2자리), MM = 문제 순번(2자리)
  출제/
    examNN_MM.c
```

예) 6장 조건문의 실습 1번 → `6장 조건문/실습/lab06_01.c`
예) 6장 조건문의 출제 2번 → `6장 조건문/출제/exam06_02.c`

- 장 폴더 이름은 자료 제목을 따른다. (예: `6장 조건문`, `10장 배열`)
- 번호는 항상 2자리(`01`, `02` …).

## 코드 스타일 (중요)

학생이 짠 코드처럼 보여야 한다. AI 티가 나면 안 된다.

- **주석 최소화.** 꼭 필요한 게 아니면 주석을 달지 않는다.
- **쉽고 단순하게.** 어렵거나 복잡한 기법, 과한 예외처리, 화려한 최적화 금지.
  수업에서 배운 범위의 평범한 문법만 쓴다.
- 파일 맨 위에 `#define _CRT_SECURE_NO_WARNINGS` 를 둔다. (scanf 경고 방지)
- 필요한 헤더만 include 한다. (`<stdio.h>` 등)
- `int main(void)` 또는 `int main()`, 끝에 `return 0;`.
- 입출력 문구는 한글로, 자료의 예시와 똑같이 맞춘다.
- 변수는 한 줄에 하나씩 단순하게 선언하는 정도면 충분하다.

참고할 만한 기존 스타일 예시는
`기초시스템프로그래밍(윤석훈)/ANU-C-Programming-Lab-master/` 안의
`chXX_solutionNN.c` 파일들에 있다.

### 스타일 예시

```c
#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

int main(void)
{
    int a;
    int b;
    int c;

    printf("3개의 정수를 입력하시오: ");
    scanf("%d %d %d", &a, &b, &c);

    int max = a > b ? a : (b > c ? b : c);
    printf("최대값: %d\n", max);
    return 0;
}
```
